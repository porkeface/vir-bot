# 个性化AI机器人（数字分身）硬件实现方案

**项目名称**：基于聊天记录蒸馏的AI机器人（模仿熟悉的人 + 硬件实体 + 视觉感知 + 多平台接入 + MCP控制）  
**作者**：hxxim（与Grok对话生成）  
**生成日期**：2026年4月21日  
**目标**：将熟悉的人的聊天记录“蒸馏”成高度仿真的AI人格，并嵌入硬件中，实现实体机器人形态，支持实时语音、多平台聊天、摄像头视觉感知、MCP外设控制。

## 1. 实现要求 & 目的

### 核心需求
- **AI人格蒸馏**：使用现有聊天记录（微信/QQ/豆包等）训练/角色化AI，让机器人“像那个人一样说话、撒娇、记得细节”。
- **硬件嵌入**：做成实体机器人（桌面/便携），支持WiFi联网。
- **MCP能力**：AI可直接调用工具控制外设（舵机、LED、电机、智能家居等）。
- **多平台接入**：支持Discord、微信、QQ（文字+语音）。
- **视觉感知**：内置摄像头，能“看到”外部环境（人脸、动作、场景），并做出自然反应。
- **语音交互**：实时语音对话 + 离线唤醒。
- **隐私安全**：所有数据本地处理（聊天记录不上云）。

### 目的
- 打造一个“活的数字分身”，兼具情感陪伴（星黎AI女友风格）与实用功能。
- 比纯软件版更具沉浸感（实体 + 视觉 + 动作）。
- 成本控制在150-270 SGD（新加坡可买），适合个人DIY。

## 2. 实现架构（文字示意图）

```text
用户（微信/QQ/Discord语音/文字） 
   ↓（WiFi）
【Raspberry Pi 5 大脑】
├── kirara-ai（或AIRI/SillyTavern）← 多平台接入 + 工作流
├── 蒸馏LoRA模型（Qwen2.5-7B/14B量化）← 你的聊天记录训练
├── 视觉模块（Qwen-VL / LLaVA）← 摄像头图像描述 + 环境感知
├── MCP工具调用 ← 通过MQTT/WebSocket发指令
   ↓（WiFi/MQTT）
【ESP32-S3 AI Cam 身体】
├── 实时语音（ASR/TTS + 离线唤醒）
├── 摄像头（实时拍图/视频）
├── MCP协议（控制舵机、LED、电机、智能家居）
├── WiFi（联网 + 传图/指令）
├── OLED/小屏显示表情
└── 电池供电（便携）
```

**通信方式**：MQTT + MCP协议（ESP32 ↔ Pi 双向指令）。

## 3. 实现方案

### 3.1 AI人格蒸馏方案
1. **简单路径（推荐先做）**：聊天记录 → 角色卡 + RAG知识库，直接加载到kirara-ai / AIRI / SillyTavern。
2. **进阶路径（真蒸馏）**：聊天记录转Alpaca/ShareGPT格式 → LoRA微调Qwen2.5 / DeepSeek → 量化部署到Pi（Unsloth / Llama-Factory / 阿里百炼）。
3. **语音克隆**（可选）：用Fish Speech / CosyVoice克隆几分钟音频，让声音也像那个人。

### 3.2 硬件方案
- **混合方案（推荐）**：Raspberry Pi 5（大脑） + ESP32-S3 AI Cam（身体）。
- **简化方案**：纯ESP32-S3 AI Cam（预算低，但微信/QQ接入稍弱）。
- **视觉处理**：ESP32摄像头拍图 → MQTT传Pi → 视觉LLM描述 → 蒸馏AI反应 + MCP动作。

### 3.3 软件栈
- ESP32固件：**xiaozhi-esp32**（最新版支持AI Cam + MCP）。
- Pi大脑：**kirara-ai**（Docker部署，多平台 + 插件）。
- 可选叠加：AIRI（动态形象）、SillyTavern（角色卡）、WeClone（专为聊天记录蒸馏）。

## 6. 上手步骤（保姆级）

1. **准备数据（蒸馏）**（1-2天）
    - 导出聊天记录 → 用WeClone或Python脚本转成Alpaca格式。
    - 云训LoRA（阿里百炼/火山引擎，几块钱）或本地Pi训（Unsloth，显存够就行）。
    - 生成角色卡 + RAG知识库（你的对话细节全记住）。
2. **硬件组装**（半天-1天）
    - ESP32-S3 AI Cam 接MIC、扬声器、舵机、电池。
    - Pi5 + 摄像头（或直接用ESP32的Cam）。
    - 3D打印外壳（可选）。
3. **刷固件 & 配置**（1天）
    - ESP32刷 xiaozhi-esp32 最新固件（GitHub有教程，支持AI Cam）。
    - WiFi配网 → 登录 xiaozhi.me 控制台绑定模型API（DeepSeek/Qwen，国内便宜）。
    - Pi装Docker → 跑 kirara-ai，配置微信/QQ/Discord（OneBot插件）。
    - 配置MQTT broker（EMQX或Mosquitto），让Pi和ESP32互通MCP指令。
4. **集成视觉 + MCP**（1-2天）
    - ESP32摄像头定期/触发拍图 → 通过MQTT发Pi。
    - Pi用视觉模型分析（“看到你在笑”）→ 喂给蒸馏AI → 生成回复 + MCP工具调用（“转头看我”“开灯”）。
    - xiaozhi-esp32 原生支持MCP工具发现与执行（舵机、GPIO、智能家居）。
5. **测试 & 优化**
    - 语音唤醒 → 多平台聊天 → 看到你进门自动问候（视觉触发）。
    - 调试Prompt：让AI用你熟悉的人的语气 + 撒娇风格。
    - 部署成开机自启动（systemd）。

## 7. 注意事项 & 后续扩展

- **隐私**：所有模型本地运行，聊天记录不上云。
- **成本控制**：先做纯ESP32简化版测试。
- **扩展方向**：加ROS2做移动机器人、叠加Live2D动态形象、语音克隆、更多外设。
- **风险**：硬件DIY需基础焊接/刷固件能力；情感陪伴仍为模拟，建议现实中多与真人互动。
