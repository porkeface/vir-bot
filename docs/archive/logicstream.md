  1. 项目解决的核心痛点

  ┌────────────────┬──────────────────────────────────────────────────────────────┐
  │      痛点      │                             说明                             │
  ├────────────────┼──────────────────────────────────────────────────────────────┤
  │ AI 失忆问题    │ 通用 AI 助手每次对话都是全新开始，无法积累与用户的共同记忆   │
  ├────────────────┼──────────────────────────────────────────────────────────────┤
  │ 人格缺失       │ 缺乏基于聊天记录蒸馏的个性化人格，无法成为真正的"数字分身"   │

  ├────────────────┼──────────────────────────────────────────────────────────────┤
  │ 记忆质量无保障 │ 缺乏写入质量控制、重复检测、冲突消解机制，记忆库会随时间劣化 │

  多 Agent 协作设计：
  - RetrievalRouter Agent：LLM 驱动的用户意图理解 + 记忆源路由
  - MemoryWriter Agent：LLM 从对话中提取结构化记忆操作
  - QualityGate Agent：规则 + LLM 双重质量把关
  - Janitor Agent：后台定时任务，自主执行衰减/合并/归档
  - 评测 Agent：自动跑分并对比历史，驱动系统进化

  开发过程也深度使用 Claude Code Agent 体系：
  通过 planner → tdd-guide → code-reviewer → security-reviewer 的多 Agent 协作流水线完成开发，每个 Phase
  均有对应的评测分数验证。
